## 一、项目概述

### 1.1 项目背景

本项目为一个**前后端分离的水质污染物数据分析平台**，采用 Spring Boot + Vue 3 的主流技术栈构建。系统旨在对水质污染物数据进行多维度分析，涵盖污染物统计、异常值检测、安全性评估、比例分析以及关联规则挖掘等功能。

### 1.2 项目定位

| 定位维度 | 描述 |
|---------|------|
| **业务领域** | 环境监测与数据分析 |
| **目标用户** | 水质监测分析人员、环境监管机构 |
| **系统类型** | 数据可视化与分析平台 |
| **部署方式** | Web 端应用（前后端分离）|

### 1.3 技术栈概览

## 二、目标分析

### 2.1 业务目标

| 序号 | 业务目标 | 描述 |
|------|---------|------|
| 1 | 污染物分布分析 | 展示不同污染物的数值分布情况（五数分布：最小值、Q1、中位数、Q3、最大值） |
| 2 | 异常值检测 | 识别并统计各污染物的异常值数量，支持异常数据筛选 |
| 3 | 安全性评估 | 评估样本的安全/非安全比例，提供安全率统计 |
| 4 | 超标比例分析 | 分析各物质的超标次数和超标率 |
| 5 | 关联规则挖掘 | 发现污染物之间的关联关系（支持度、置信度、提升度） |
| 6 | 规则查询 | 提供灵活的条件查询功能，支持按规则指标筛选 |

### 2.2 技术目标

| 序号 | 技术目标 | 描述 |
|------|---------|------|
| 1 | 前后端分离架构 | 实现前后端独立开发、部署，提高开发效率 |
| 2 | RESTful API 设计 | 提供规范化的 REST 接口，便于前端调用和集成 |
| 3 | 数据可视化 | 使用 ECharts 实现丰富的图表展示（箱线图、柱状图、饼图、散点图） |
| 4 | 请求缓存机制 | 前端实现 5 分钟请求缓存，减少重复请求 |
| 5 | 跨域支持 | 配置 CORS 支持前端多端口开发 |

## 五、问题与解决方案

### 5.1 已识别问题

#### 5.1.1 安全风险

| 序号 | 问题描述 | 严重程度 | 问题位置 |
|------|---------|---------|---------|
| 1 | 数据库密码 `123456` 明文存储在 `application.properties` | ⚠️ 高 | `backend/src/main/resources/application.properties` |
| 2 | 无认证授权机制，任何人都可访问 API | ⚠️ 高 | 全局 |
| 3 | CORS 配置开放多个 localhost 端口 | 🟡 中 | `CorsConfig.java` |
| 4 | 生产环境可能暴露 SQL 日志 | 🟡 中 | `application.properties` |

#### 5.1.2 代码质量问题

| 序号 | 问题描述 | 严重程度 | 问题位置 |
|------|---------|---------|---------|
| 1 | Dashboard.vue 中直接使用 `http://localhost:8080` 硬编码 | ⚠️ 中 | `Dashboard.vue` 第 352、366、378 行 |
| 2 | `FindRulevantRules` 疑似拼写错误（应为 FindRelevantRules） | △ 低 | `FindRulevantRules.java` |
| 3 | 多个 API 端点直接拼接 URL | △ 中 | 多个 Vue 组件 |
| 4 | 缺少全局异常处理器 | △ 中 | 全局 |

#### 5.1.3 性能问题

| 序号 | 问题描述 | 严重程度 | 问题位置 |
|------|---------|---------|---------|
| 1 | 数据库表缺少索引 | 🟡 中 | 数据库层 |
| 2 | ECharts 全量引入，未做 tree-shaking | △ 低 | `package.json` |
| 3 | 无 HTTP 缓存头配置 | △ 低 | 后端响应 |

### 5.2 解决方案

#### 5.2.1 安全加固方案

| 问题 | 解决方案 | 实施难度 |
|------|---------|---------|
| 数据库密码明文 | 使用环境变量 + Jasypt 加密 | 中等 |
| 无认证机制 | 引入 JWT + Spring Security | 较高 |
| CORS 开放端口 | 生产环境配置实际域名 | 简单 |
| SQL 日志暴露 | 生产环境关闭 `show-sql` | 简单 |

**数据库密码加密配置示例：**

```properties
# 使用环境变量
spring.datasource.password=${DB_PASSWORD}

# 或使用 Jasypt 加密
spring.datasource.password=ENC(加密后的密文)
```

#### 5.2.2 代码质量改进方案

| 问题 | 解决方案 | 实施难度 |
|------|---------|---------|
| URL 硬编码 | 统一使用 `request.js` 配置的 baseURL | 简单 |
| 命名拼写错误 | 重命名 `FindRulevantRules` → `FindRelevantRules` | 简单 |
| 缺少全局异常处理 | 添加 `@ControllerAdvice` 全局异常处理器 | 中等 |

**全局异常处理器示例：**

```java
@RestControllerAdvice
public class GlobalExceptionHandler {
    @ExceptionHandler(Exception.class)
    public Result<?> handleException(Exception e) {
        log.error("系统异常", e);
        return Result.error("系统繁忙，请稍后重试");
    }
}
```

#### 5.2.3 性能优化方案

| 优化项 | 实施方案 | 预期收益 |
|--------|---------|---------|
| 数据库索引 | 为 `pollutant_name`、`is_safe` 等字段添加索引 | 查询性能 10x 提升 |
| ECharts 按需加载 | 使用 `echarts/core` 按需引入图表类型 | bundle 减少 30% |
| Redis 缓存 | 引入 Redis 缓存热点数据 | 响应时间减少 50% |
